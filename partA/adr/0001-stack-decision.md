# ADR-001: Технологийн стак болон хөгжүүлэлтийн орчны сонголт

## Төлөв байдал
Зөвшөөрөгдсөн (Accepted) - *2026-05-09-нд батлав*

## Контекст (Context)
ШУТИС-ийн МХТС-ийн "CSA311: Програм хангамжийн чанар ба найдвартай байдал" хичээлийн Lab13 бие даалтын хүрээнд "Mini Library" системийг хөгжүүлэх шаардлага гарсан. Төсөлд тавигдах үндсэн шаардлага нь:
1. **Robustness:** Lec12-ын зарчмын дагуу алдаанд тэсвэртэй байх.
2. **Testability:** 10-аас доошгүй нэгж тест (Unit Test) ажилладаг байх.
3. **AI Collaboration:** AI туслахтай (Claude) үр дүнтэй хамтран ажиллах.

## Шийдвэр (Decision)
Бид төслийн эхний шатанд дараах технологийн стакыг үндсэн сонголтоор тодорхойлсон:

* **Backend:** Java 17 / Spring Boot 3.2.5
    * *Үндэслэл:* Enterprise түвшний найдвартай ажиллагаа болон Dependency Injection (DI) механизмыг ашиглан кодыг модульчлах.
* **Database:** H2 (In-memory)
    * *Үндэслэл:* Суулгах, тохируулах шаардлагагүй (Zero-config), unit тест ажиллуулах явцад өгөгдлийг түр хадгалахад хамгийн хурдан шийдэл.
* **Productivity:** Project Lombok
    * *Үндэслэл:* Java-ийн нүсэр кодын (Boilerplate) хэмжээг багасгаж, логик хэсэгт илүү анхаарах.
* **Build Tool:** Maven
    * *Үндэслэл:* Хамааралтай сангуудыг (Dependencies) автоматаар удирдах болон багшийн шаардсан `target/` бүтцийг үүсгэх.

## Үр дүн (Consequences)
* **Эерэг:** Төслийн бүтэц стандарт (standardized) болж, хөгжүүлэлтийн эхний суурь амжилттай тавигдсан.
* **Сөрөг:** Spring Boot-ийн тохиргоо нүсэр байгаа нь хөнгөн хэмжээний API хөгжүүлэхэд хугацаа алдах эрсдэлтэй. 
* **Тэмдэглэл:** Хөгжүүлэлтийн явцад AI-тай хамтран ажиллах хурдыг нэмэгдүүлэх зорилгоор дараагийн ADR-аар (ADR-002) стак шинэчлэх боломжийг нээлттэй үлдээв.
