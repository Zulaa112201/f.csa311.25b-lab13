# ADR-002: Java/Spring Boot-өөс Node.js/TypeScript рүү шилжсэн шийдвэр

## Төлөв байдал
Зөвшөөрөгдсөн (Accepted) - *2026-05-11-нд батлав*

## Контекст (Context)
Төслийн эхний шатанд (ADR-001) Java/Spring Boot ашиглахаар төлөвлөсөн боловч хөгжүүлэлтийн явцад дараах хүндрэлүүд гарсан:
1. **Context Management:** AI (Claude)-тай хамтран ажиллах явцад Spring Boot-ийн нүсэр бүтэц (Controller, Service, Repository, DTO, Entity) нь контекстыг хэт томруулж, AI-ийн хариултын нарийвчлалыг бууруулж байсан.
2. **Setup Overhead:** Бага хэмжээний "Mini Library" системд Spring Boot-ийн тохиргоо (Maven, JRE, XML/Annotations) нь хэт нүсэр (Over-engineering) байв.
3. **Robustness Implementation:** Lec12-ын алдааг барих логикийг TypeScript-ийн хөнгөн асинхрон (Async/Await) бүтэц дээр хэрэгжүүлэх нь илүү уян хатан байсан.

## Шийдвэр (Decision)
Бид төслийн үндсэн технологийг **Node.js + Express + TypeScript** болгон өөрчлөх шийдвэр гаргасан.
- **TypeScript:** Java-тай адил Strong Typing-ийг хангана.
- **SQLite:** H2-ийн оронд илүү хөнгөн, файл дээр суурилсан сан ашиглана.
- **Jest:** Төслийн unit тестүүдийг илүү хурдан ажиллуулах зорилгоор.

## Үндэслэл (Rationale)
* **AI-Assisted Efficiency:** TypeScript-ийн нэг файл дээр логикоо багцлах боломж нь AI-тай хамтран ажиллахад 2 дахин хурдан байсан.
* **Type Safety:** `interface` ашиглан "Robustness" шаардлагыг бүрэн хангасан.
* **Portability:** Багшийн орчинд `npm install` хийгээд л шууд ажиллах боломжтой (JRE-ийн хамааралгүй болсон).

## Үр дүн (Consequences)
* **Эерэг:** Код бичих хурд нэмэгдэж, 20+ unit тестүүдийг маш богино хугацаанд бичиж дуусгасан.
* **Эерэг:** Төслийн бүтэц илүү ойлгомжтой, "Lec12-Robustness" зарчмыг хэрэгжүүлэхэд хялбар болсон.
* **Сөрөг:** Өмнө нь бичигдсэн Java кодуудыг устгах шаардлага гарсан.
