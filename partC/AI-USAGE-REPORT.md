# AI Usage Report - Mini Library System

## 1. Юуг AI хийсэн, юуг өөрөө хийсэн?
Энэхүү төслийн явцад AI (Claude) болон хүний оролцоог дараах байдлаар хуваарилсан:

* **AI-ийн гүйцэтгэсэн ажил:**
    * Системийн суурь бүтэц (Spring Boot Entity, Repository, Service) болон boilerplate кодуудыг үүсгэсэн.
    * Maven-ийн `pom.xml` файл дээр гарсан dependency-ийн алдаануудыг оношилж, засах кодоор хангасан.
    * 10 гаруй unit тестүүдийн ерөнхий логик болон бүтцийг санал болгосон.
    * OpenAPI 3.0 спецификаци болон Custom Slash Command-уудын загварыг гаргасан.

* **Өөрийн (Миний) гүйцэтгэсэн ажил:**
    * AI-аас гарсан кодуудыг өөрийн IDE (VS Code) дээр ажиллуулж, гарсан улаан зураас болон сангуудын алдааг гараар зассан.
    * Төслийн хавтасны бүтцийг багшийн удирдамжийн дагуу (partA, partB, partC) цэгцэлж, файлуудыг зөв байршилд шилжүүлсэн.
    * AI-ийн санал болгосон логик (жишээ нь LoanService дээрх availability check) миний хүсэж буй "Mini Library" системийн шаардлагад нийцэж байгаа эсэхийг хянаж, баталгаажуулсан.
    * Git workflow-ыг удирдан, commit бүрийг утга учиртай нэрлэж, AI-ийн оролцоог тэмдэглэсэн.

## 2. Hallucination (AI-ийн төөрөгдөл) 2+ жишээ
AI-тай ажиллах явцад дараах 2 тохиолдолд буруу мэдээлэл эсвэл алдаатай код өгсөн:

1. **Dependency Hallucination:** AI эхний ээлжинд `jakarta.persistence.*` сангуудыг ашиглах код өгсөн боловч миний `pom.xml` файл дээр эдгээр сангууд суугаагүй байсан. Үүнээс болж бүх код улаан зураастай болж алдаа заасан. Би үүнийг AI-аас дахин асууж, Maven reload хийх заавар авч зассан.
2. **Directory Structure Error:** AI надад кодыг бичиж өгөхдөө шууд `src/main/java...` гэсэн зам зааж өгсөн боловч багшийн удирдамжид энэ нь заавал `partB/src/...` дотор байх ёстой байсан. Хэрэв би AI-д шууд итгэсэн бол багшийн тавьсан хавтасны бүтцийн шаардлагыг зөрчих байлаа. Би үүнийг өөрөө гараар хавтас үүсгэж шилжүүлсэн.
